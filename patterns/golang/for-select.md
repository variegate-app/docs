# for-select

## [<<< ---](../index.md)

Один из базовых паттернов. Как правило используется для чтения данных из нескольких каналов.

```go
var c1, c2 <-chan int

for { // либо бесконечный цикл, либо range по чему‑то
    select {
    case <-c1: // делаем какую‑то работу с каналами
    case <-c2:
    default: // выполняется, если остальные ветки не готовы
    }

    // дополнительная работа
}
```

Оператор `select` синтаксически похож на `switch`, но ведёт себя по‑другому. Все ветки `case` рассматриваются одновременно и имеют **равные шансы** быть выбраны. Если ни один из `case` не готов к выполнению, весь оператор `select` блокируется.

## Типовые ошибки / антипаттерны
- Делать `default` в цикле, не добавив sleep/backoff: получаете busy-wait и перегруз CPU.
- Использовать `select` без стратегии остановки: в итоге goroutine “живет вечно”.
- Не предусматривать, что каналы могут быть закрыты (`ok == false`), и продолжать читать “нулевые” значения.

## Практический чеклист
- Есть план выхода из цикла `for` (отмена `done`/`ctx.Done()` или закрытие каналов).
- В ветках чтения учитывается `ok` при чтении из канала.
- Если нужен non-blocking режим — `default` присутствует осознанно (частота ограничена).
- Для блокирующих операций используется таймаут или отменяемый контекст там, где это важно.

## Как адаптировать под кейс
- Если вы читаете из нескольких каналов и важна реакция “к первому готовому”, `select` — базовая техника.
- Если нужен fair-распределение нагрузки, обеспечьте одинаковую семантику веток (и одинаковые условия готовности).
