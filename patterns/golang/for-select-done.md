# for-select-done

## [<<< ---](../index.md)

Горутины не собираются сборщиком мусора автоматически, поэтому могут легко "утечь".

```go
go func() {
// <операция, которая будет блокироваться вечно>
// => горутина утекает
}()
// выполняем работу

```

Чтобы избежать утечек, горутину нужно отменять, когда ей об этом сигнализируют. Родительская горутина должна послать сигнал отмены дочерней через *канал только для чтения* с именем `done`. По соглашению его передают первым параметром.

Этот паттерн активно используется и в других паттернах.

```go
// дочерняя горутина
doWork(<-done chan interface {}, other_params) <- terminated chan interface{} {
    terminated := make(chan interface{}) // сообщает внешнему коду, что работа завершена
    defer close(terminated)

    for {
        select: {
            case: // выполняем основную работу
            case <- done:
                return
        }
        // дополнительная работа
    }

    return terminated
}

// родительская горутина
done := make(chan interface{})
terminated := doWork(done, other_args)

// делаем что‑то
// затем говорим дочерней горутине остановиться
close(done)

// ждём, пока дочерняя горутина завершит работу
<-terminated
```

## Типовые ошибки / антипаттерны
- Отменять горутину, но не ожидать её завершения: получаются гонки и неожиданные побочные эффекты.
- Использовать канал `done` не по договорённости (вместо `<-chan struct{}` / read-only параметра), повышается шанс ошибочного закрытия.
- Дочерняя горутина игнорирует сигнал остановки и продолжает работу.

## Практический чеклист
- Родительская горутина передаёт `done` как channel-only для чтения (`<-done` в параметрах).
- В дочерней горутине `select` включает ветку `case <-done: return`.
- Для “кто завершился” есть сигнал/канал (`terminated`) или другое ожидание.
- `done` закрывается (или отправляется сигналом) ровно один раз координатором.

## Как адаптировать под кейс
- Если отмена должна иметь дедлайн — переходите на `context.Context` поверх `done`.
- Если требуется отмена “группой” — рассмотрите `errgroup` или `graceful shutdown`.
