# Одиночка (Singleton)

## [<<< ---](../../index.md)

## Назначение

Singleton гарантирует, что в системе существует ровно один экземпляр класса, и предоставляет глобальную точку доступа к нему.

## Когда использовать

- когда единственный экземпляр действительно должен управлять некоторым глобальным ресурсом (например, кэш, логгер, конфигурация);
- когда создание экземпляра дорого и его разумно переиспользовать;
- когда важно строго контролировать жизненный цикл.

## Предупреждение

Singleton часто ухудшает тестируемость из-за глобального состояния. В современных системах нередко предпочитают dependency injection или явные менеджеры ресурсов.

## Пример реализации на Go

```go
package main

import (
	"fmt"
	"sync"
)

type Config struct {
	Env string
}

var (
	once     sync.Once
	instance *Config
)

func GetConfig() *Config {
	once.Do(func() {
		instance = &Config{Env: "production"}
	})
	return instance
}

func main() {
	c1 := GetConfig()
	c2 := GetConfig()

	fmt.Println(c1.Env, c1 == c2)
}
```
