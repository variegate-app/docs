# Hash Table

## [<<< ---](../index.md)

Hash Table (хеш-таблица) хранит пары ключ-значение и дает быстрый доступ по ключу в среднем случае.

Ключевая идея: через хеш-функцию ключ сразу попадает в корзину, поэтому `Search/Insert/Delete` обычно `O(1)`, но при коллизиях возможен деградирующий `O(n)`.

## Аналогия из жизни

Картотека с ячейками по буквам: по фамилии сразу идем в нужный отсек, а внутри отсека смотрим карточки.

## Когда применять

- Частый поиск/обновление по ключу.
- Кэширование (map key -> value).
- Дедупликация и подсчет частот.

## Когда лучше выбрать другую структуру

- Нужен упорядоченный обход: `./binary-search-tree.md` или `./avl-tree.md`.
- Нужен диапазонный поиск: `./b-tree.md` или `./kd-tree.md`.
- Нужны гарантии worst-case `O(log n)`: `./red-black-tree.md`.
