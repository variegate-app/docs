# B-Tree

## [<<< ---](../index.md)

B-Tree - сбалансированное многопутевое дерево поиска, оптимизированное для блочного хранения (диск/SSD).

Ключевая идея: уменьшается глубина дерева, поэтому `Search/Insert/Delete` остаются `O(log n)` при минимуме чтений страниц.

## Аналогия из жизни

Картотека по шкафам и ящикам: в каждом ящике много отсортированных карточек, поэтому глубина переходов маленькая.

## Когда применять

- Индексы СУБД.
- Файловые системы.
- Большие наборы данных, не помещающиеся в RAM.

## Когда лучше выбрать другую структуру

- Данные в памяти и нужен простой код: `./avl-tree.md` или `./red-black-tree.md`.
- Нужен максимально быстрый lookup без порядка: `./hash-table.md`.
- Нужна структура приоритетов: `./heap.md`.
