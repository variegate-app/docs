# KD Tree

## [<<< ---](../index.md)

KD Tree - k-мерное дерево для пространственных данных (точки, координаты, nearest-neighbor).

Ключевая идея: дерево делит пространство по осям, поэтому поиск/вставка обычно `O(log n)` в среднем, но зависит от распределения данных.

## Аналогия из жизни

Поиск ближайшего кафе на карте: сначала делим карту по районам, затем уточняем внутри района.

## Когда применять

- Геоданные и карты.
- Поиск ближайших соседей (k-NN).
- Пространственная индексация в 2D/3D.

## Когда лучше выбрать другую структуру

- Нужны только ключ-значение операции: `./hash-table.md`.
- Нужен одномерный упорядоченный индекс: `./avl-tree.md` или `./red-black-tree.md`.
- Данные очень динамичны и плохо балансируются: `./b-tree.md` или `./graph.md` (по задаче).
