# Splay Tree

## [<<< ---](../index.md)

Splay Tree - самонастраивающееся бинарное дерево поиска, где часто используемые элементы поднимаются к корню.

Ключевая идея: отдельная операция может быть дорогой, но амортизированно последовательность операций дает `O(log n)`.

## Аналогия из жизни

Документы на столе: те, к которым обращаются чаще, кладут сверху для более быстрого доступа.

## Когда применять

- Есть "горячие" ключи, которые часто повторяются.
- Подходит для кэширующих/адаптивных сценариев.
- Важна простота без хранения дополнительной баланс-информации.

## Когда лучше выбрать другую структуру

- Нужны жесткие worst-case гарантии на каждую операцию: `./red-black-tree.md` или `./avl-tree.md`.
- Нужен очень быстрый lookup без порядка: `./hash-table.md`.
- Нужна дисковая индексация: `./b-tree.md`.
