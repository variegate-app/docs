# Tim sort

## [<<< ---](../index.md)

Tim sort - гибридный алгоритм, который сочетает идеи insertion sort и merge sort и оптимизирован для реальных данных.

Ключевая идея: алгоритм ищет уже отсортированные фрагменты (runs), локально улучшает их insertion sort и сливает по правилам merge sort.

## Принцип работы

1. Находим естественные runs (уже упорядоченные последовательности).
2. Короткие runs доводим insertion sort до минимальной длины.
3. Сливаем runs, поддерживая баланс слияний.
4. Получаем стабильную сортировку с хорошей практической производительностью.

## Когда применять

- Нужна стабильная сортировка в production.
- Данные часто частично отсортированы.
- Важна хорошая практическая скорость, а не только асимптотика.

## Когда лучше выбрать другой алгоритм

- Нужна максимально простая реализация: `./merge-sort.md` или `./quick-sort.md`.
- Данные полностью случайные и важна низкая константа в памяти: `./quick-sort.md`.
- Специальный целочисленный кейс: `./radix-sort.md` или `./counting-sort.md`.
